# 机器人维修站：闯关必备知识点


## 1. 面向对象的核心思路

面向对象编程会把程序中的事物看成一个个对象。对象通常有两类内容：

- 属性：对象“有什么”，例如名字、型号、能量、任务列表。
- 方法：对象“能做什么”，例如充电、添加任务、完成任务、生成报告。

类是模板，实例是根据模板创建出来的具体对象。

```python
class 类名:
    pass

对象名 = 类名()
```

类名一般使用大驼峰命名法，例如 `RepairBot`、`TaskBoard`、`DispatchCenter`。

## 2. 定义类和创建实例

定义类使用 `class` 关键字。类体内部的代码要缩进。

```python
class Robot:
    pass

robot = Robot()
```

创建实例时，本质是调用类名。每调用一次类名，就会创建一个新的实例对象。

```python
robot1 = Robot()
robot2 = Robot()
```

`robot1` 和 `robot2` 是两个不同对象，即使它们由同一个类创建，也可以拥有不同的数据。

## 3. `__init__` 初始化方法

`__init__` 是初始化方法。创建实例时，Python 会自动调用它。

常见格式：

```python
class Robot:
    def __init__(self, name, energy):
        self.name = name
        self.energy = energy
```

这里有几个关键点：

- `__init__` 的名字固定，前后都是两个下划线。
- 第一个参数通常写 `self`，表示当前正在创建的实例对象。
- `self.name = name` 表示给当前实例添加 `name` 属性。
- 创建实例时，不需要手动传入 `self`。

```python
robot = Robot('小蓝', 80)
```

这行代码会把 `'小蓝'` 传给 `name`，把 `80` 传给 `energy`。

## 4. 实例属性

通过 `self.属性名 = 值` 保存到实例身上的属性，叫实例属性。

```python
class Robot:
    def __init__(self, name, energy):
        self.name = name
        self.energy = energy
```

访问实例属性使用点语法：

```python
robot = Robot('小蓝', 80)
print(robot.name)
print(robot.energy)
```

修改实例属性也使用点语法：

```python
robot.energy = 60
```

实例创建后，也可以追加新属性：

```python
robot.status = '工作中'
```

查看一个实例身上的所有实例属性，可以使用：

```python
print(robot.__dict__)
```

闯关中经常会要求把 `对象.__dict__` 保存到某个变量，目的是检查你是否正确创建了实例属性。

## 5. 多个实例互不干扰

每个实例都有自己独立的一份实例属性。

```python
class Robot:
    def __init__(self, name, energy):
        self.name = name
        self.energy = energy

r1 = Robot('小蓝', 80)
r2 = Robot('小绿', 90)

r1.energy = 50
```

修改 `r1.energy` 不会影响 `r2.energy`。如果题目要求“只修改某一个对象”，要确认自己没有改到类属性或另一个实例。

## 6. 实例方法

定义在类中的普通函数，通常叫实例方法。实例方法的第一个参数通常是 `self`。

```python
class Battery:
    def __init__(self, energy):
        self.energy = energy

    def use(self, amount):
        self.energy -= amount
        return self.energy
```

调用实例方法：

```python
battery = Battery(100)
left = battery.use(20)
```

注意：

- 定义方法时要写 `self`。
- 调用方法时不需要传 `self`。
- 方法内部可以读取 `self.energy`。
- 方法内部也可以修改 `self.energy`。
- 如果题目要求保存返回值，要使用变量接住方法调用结果。

```python
result = 对象.方法名(参数)
```

## 7. `return` 和 `print` 的区别

闯关判题通常检查变量值，所以更常用 `return`。

```python
def add(a, b):
    return a + b

result = add(3, 5)
```

`print()` 只是把内容显示在终端，不会自动保存为变量。

```python
def add(a, b):
    print(a + b)

result = add(3, 5)  # result 会是 None
```

如果题目说“把结果保存到某变量”，通常要用赋值语句：

```python
变量名 = 表达式或函数调用
```

## 8. 在方法中使用条件判断

方法内部可以使用 `if`、`elif`、`else` 完成限制、分类和兜底处理。

示例：把数值限制在 0 到 100 之间。

```python
class Meter:
    def __init__(self, value):
        self.value = value

    def set_value(self, value):
        if value < 0:
            self.value = 0
        elif value > 100:
            self.value = 100
        else:
            self.value = value
        return self.value
```

也可以使用 `min()` 和 `max()` 简化数值边界：

```python
value = min(value, 100)
value = max(value, 0)
```

## 9. 在方法中使用列表

列表适合保存有顺序、可变化的一组数据。任务看板、订单列表、机器人列表都可以用列表。

创建空列表：

```python
self.tasks = []
```

添加元素：

```python
self.tasks.append(task)
```

获取列表长度：

```python
count = len(self.tasks)
```

删除并返回第一个元素：

```python
first = self.tasks.pop(0)
```

判断列表是否为空：

```python
if self.tasks:
    # 列表不为空
else:
    # 列表为空
```

如果需要保存当前列表内容，并且后面还会继续修改原列表，可以保存一份拷贝：

```python
snapshot = self.tasks.copy()
```

## 10. 类属性

类属性是直接写在类里面的赋值语句。它属于类本身，通常用于保存公共数据。

```python
class Ticket:
    max_priority = 5
    station = '主维修站'

    def __init__(self, code, priority):
        self.code = code
        self.priority = priority
```

访问类属性：

```python
print(Ticket.max_priority)
print(Ticket.station)
```

实例也能访问类属性：

```python
ticket = Ticket('A01', 3)
print(ticket.station)
```

但如果执行：

```python
ticket.station = '临时站点'
```

这会给 `ticket` 这个实例单独添加或修改一个实例属性 `station`，不会改变 `Ticket.station`。

这一点在类属性关卡中非常重要。

## 11. 类属性与实例属性的查找顺序

当写 `对象.属性名` 时，Python 会按顺序查找：

1. 先看对象自己身上有没有这个实例属性。
2. 如果对象自己没有，再去创建它的类里面找类属性。

示例：

```python
class Robot:
    planet = '地球'

r1 = Robot()
r2 = Robot()

r1.planet = '火星'
```

此时：

- `r1.planet` 读取的是 `r1` 自己的实例属性。
- `r2.planet` 读取的是 `Robot` 的类属性。
- `Robot.planet` 仍然是原来的类属性。

## 12. 类方法

类方法使用 `@classmethod` 修饰。第一个参数通常写 `cls`，表示当前类本身。

```python
class Record:
    default_status = '待处理'

    def __init__(self, name, status):
        self.name = name
        self.status = status

    @classmethod
    def create(cls, text):
        name = text
        return cls(name, cls.default_status)
```

调用类方法推荐使用类名：

```python
record = Record.create('小蓝')
```

类方法常见用途：

- 读取或修改类属性。
- 根据一段文本、字典或其他数据创建实例。
- 把“创建对象前的数据整理逻辑”封装起来。

## 13. 字符串拆分与类型转换

从文本创建对象时，常见做法是使用 `split()`。

```python
text = '小蓝-2026-3'
name, year, times = text.split('-')
```

`split('-')` 会按 `-` 把字符串拆成多个部分。

拆出来的内容都是字符串。如果题目需要数字，要转换：

```python
year = int(year)
times = int(times)
```

如果要把拆出来的数据传入类创建实例：

```python
obj = 类名(name, int(year), int(times))
```

在类方法中可以写成：

```python
return cls(name, int(year), int(times))
```

这样如果以后类名变化，类方法仍然更容易复用。

## 14. 静态方法

静态方法使用 `@staticmethod` 修饰。它不接收 `self`，也不接收 `cls`。

```python
class Tool:
    @staticmethod
    def double(n):
        return n * 2
```

调用静态方法推荐使用类名：

```python
result = Tool.double(5)
```

静态方法适合放和这个类相关、但不需要访问实例属性和类属性的小工具。例如：

- 字符串脱敏。
- 分数评级。
- 格式转换。
- 简单计算。

## 15. 字符串切片

字符串可以像列表一样使用索引和切片。

```python
code = 'ABC-1234-XY'
first_three = code[:3]
last_two = code[-2:]
```

常见规则：

- `s[:3]` 表示从开头取到索引 3 之前。
- `s[-2:]` 表示取最后两个字符。
- 字符串可以用 `+` 拼接。

```python
masked = code[:3] + '***' + code[-2:]
```

## 16. 列表推导式

如果要对一组数据逐个调用函数或方法，可以使用普通循环：

```python
levels = []
for score in scores:
    levels.append(get_level(score))
```

也可以使用列表推导式：

```python
levels = [get_level(score) for score in scores]
```

列表推导式适合写比较短、逻辑清晰的转换。刚开始学习时，用普通 `for` 循环也完全可以。

## 17. 字典

字典适合保存“键和值”的对应关系。

```python
robot = {
    'name': '小蓝',
    'model': 'R-1',
    'tasks': 3,
}
```

读取字典中的值：

```python
name = robot['name']
tasks = robot['tasks']
```

修改或新增键值对：

```python
robot['status'] = '工作中'
```

在综合关卡中，常见做法是把多个字典放进列表：

```python
robots = []
robots.append({'name': '小蓝', 'model': 'R-1', 'tasks': 3})
```

## 18. 集合

集合适合去重。

```python
models = ['R-1', 'R-2', 'R-1']
unique_models = set(models)
```

也可以在循环中逐个添加：

```python
unique_models = set()
for item in items:
    unique_models.add(item)
```

注意：集合是无序的。闯关判题会对集合做规范化处理，所以只要集合内容正确即可。

空集合必须写成：

```python
empty_set = set()
```

不能写 `{}`，因为 `{}` 表示空字典。

## 19. 循环统计

综合任务中常见的统计包括：

- 数量统计。
- 总和统计。
- 条件筛选。
- 去重收集。

累计总和：

```python
total = 0
for item in items:
    total += item['count']
```

条件筛选：

```python
selected = []
for item in items:
    if item['count'] >= 3:
        selected.append(item['name'])
```

去重收集：

```python
models = set()
for item in items:
    models.add(item['model'])
```

## 20. 在类中生成报告字典

当题目要求“返回一个报告”，通常可以创建一个字典并返回。

```python
class Center:
    def __init__(self, name):
        self.name = name
        self.items = []

    def build_report(self):
        total = 0
        for item in self.items:
            total += item['count']

        report = {
            'name': self.name,
            'item_count': len(self.items),
            'total': total,
        }
        return report
```

也可以直接返回字典：

```python
return {
    'name': self.name,
    'item_count': len(self.items),
    'total': total,
}
```

注意字典键名必须和题目要求一致。

## 21. 常见错误清单

### 21.1. 忘记缩进

类中的方法、方法中的代码都必须正确缩进。

```python
class Robot:
    def __init__(self, name):
        self.name = name
```

### 21.2. 把 `self` 写漏

定义实例方法时，通常第一个参数要写 `self`。

```python
def charge(self, amount):
    ...
```

### 21.3. 调用方法时手动传入 `self`

调用实例方法时，不需要写 `self`。

```python
robot.charge(20)
```

### 21.4. 只定义了类，没有创建实例

题目如果要求保存实例信息、调用方法或生成报告，通常必须先创建实例。

```python
obj = 类名(参数)
```

### 21.5. 方法写了 `return`，但没有保存返回值

```python
result = obj.method()
```

### 21.6. 变量名和题目要求不一致

判题会检查指定变量。变量名大小写、下划线都要完全一致。

### 21.7. 把类属性误改成实例属性

`对象.属性 = 值` 修改的是对象自己身上的属性，不会直接修改类属性。

如果要修改类属性，通常使用：

```python
类名.属性 = 值
```

### 21.8. 忘记把字符串数字转成整数

`split()` 拆出来的都是字符串，需要时要用 `int()` 转换。

```python
number = int(text_number)
```

## 22. 闯关前自查

提交前可以按下面顺序检查：

1. 是否定义了题目要求的类名。
2. `__init__` 的参数是否和题目要求一致。
3. 实例属性是否都通过 `self.属性名` 保存。
4. 是否创建了题目要求的实例对象。
5. 是否调用了题目要求的方法。
6. 方法是否返回了题目需要的结果。
7. 是否把结果保存到了指定变量名。
8. 列表、字典、集合的内容结构是否和题目描述一致。
9. 类属性和实例属性是否区分清楚。
10. 代码是否能从上到下正常运行，没有语法错误。

## 23. 建议的解题节奏

每一关可以按这个流程写：

1. 先读题，圈出类名、方法名、属性名、结果变量名。
2. 写出类的空框架。
3. 补上 `__init__` 和实例属性。
4. 补上方法逻辑。
5. 创建实例并调用方法。
6. 把结果保存到题目要求的变量。
7. 运行代码，查看报错或变量内容。

不要一上来追求一行写完。面向对象题更看重结构清楚：谁拥有数据，谁负责行为，最后结果保存在哪里。
